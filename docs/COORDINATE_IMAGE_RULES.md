# 暫定ルール：記事用コーディネート画像

このファイルは、`ILLUSTRATION_GUIDE.md` に追記した暫定ルールのリポジトリ内控えです。

## 基本
- 記事用CSVで指定されたメインカラー、サブカラー、アクセントカラーを使う。
- すべての記事には、必ず1枚以上の記事用画像を入れる。
- 記事一覧・関連記事などで使うサムネイルには、その記事内で使用している記事用画像を使う。
- 記事画像は、3D AIモデル風のコーディネート写真とカラーパネルを1枚の画像として生成する手順を基本とする。
- コードだけで作ったマネキン風・図解風の仮画像は、完成版の記事画像として扱わない。
- リアルな写真に近づけるため、生成プロンプトで35mmフィルム調の細かな粒子感・自然なざらつきを指定する。
- 最終合成時にも、写真部分へ控えめなフィルムグレインを重ね、AI生成特有の過度な滑らかさを抑える。
- 4つのコーディネート写真を2x2で隣接配置する。中央に空白帯は作らない。
- 4つの写真の境界線は、画像の正確な中心で交差させる。1024x1536の縦画像では、左右の境界線を `x = 512`、上下の境界線を `y = 768` に通す。
- 左右2列は同じ幅、上下2行は同じ高さにする。写真ごとの構図の崩しは各象限内で行い、2x2グリッド自体はずらさない。
- 中央の白い境界線は細く保つ。4枚の写真の間に大きな空白や、中心からずれた交差点を作らない。
- 4つのコーディネート写真はすべて縦長3:2で統一する。
- 4つの写真は、立つ、座る、前から、斜めから、後ろから、ポーズあり、ポーズなしなど、構図に変化をつける。
- 4つの写真は、すべて違うモデルをイメージして生成する。体型、身長感、肌の見える範囲、髪の長さや髪色、着こなしの雰囲気が同一人物に見えないように散らす。
- 4つの写真は、背景や光から成る世界観もできるだけ散らす。屋外のまま、街路、カフェ前、駅前、階段、公園沿い、夕方の逆光、曇天の柔らかい光、晴天の硬めの影などを混ぜる。
- 4つのコーディネートで同じキーアイテムを使い回さない。
- シーンはすべて屋外にする。

## 人物表現
- 3D AIモデル風の人物表現を使う。
- 口元以上は完全にトリミングし、口、鼻、目、額、顔全体が見えない構図にする。
- キャラクター本人に見える髪型、顔立ち、固有小物、公式衣装、ロゴ、紋章、固有マークは使わない。

## Aモデル固定・実写寄せ生成基準
- Aモデルを採用する記事用画像では、成人日本人女性のAモデルを同一人物として固定する。
- Aモデル採用時は、基本ルールの「4つの写真はすべて違うモデルをイメージして生成する」よりもAモデル固定を優先する。同一人物に見せたまま、服・構図・光・背景で変化を出す。
- Aモデルの基準は、20代前半、背が高く細身、自然な黒髪ロング、前髪あり、細長い輪郭、落ち着いた表情、淡いナチュラルメイクとする。
- Aモデル固定画像では、顔出しを許可する。顔を出す場合でも、公式キャラクター本人、著名人、実在人物、アニメ風、コスプレ風には寄せない。
- 初音ミクなど既存キャラクターをテーマにする場合も、表現するのは色・雰囲気・コーデの方向性だけにする。公式キャラクター、公式衣装、ロゴ、スクショ、固有記号、ツインテール、髪色コピーは使わない。
- 基準候補画像は `/Users/kensuke_kawamura/.codex/generated_images/019f32e4-c89f-7ed3-aab8-56c945181885/ig_014f7c9f844931f6016a4e4fac6d74819198734e0ea372014.png` とする。街角に寄せすぎた次版ではなく、この1つ前の質感・光・整いすぎないバランスを基準にする。
- 最優先品質は「AI生成感があるか」と聞かれて「ない」と言えること。美しさ、均整、クリーンさより、実写スナップとしての自然さを優先する。
- 肌と脚は滑らかにしすぎない。毛穴、薄い色むら、微細な影、膝・すね・足首の自然な質感、靴下や靴による軽い圧迫跡、乾燥感を入れる。
- 全体に偶然性を入れる。控えめなフィルムグレイン、影部のセンサーノイズ、髪の乱れ、服のしわ、黒い服の小さなlint、軽い毛羽立ち、靴の擦れ、舗装の汚れを許容する。
- 手元、服の端、アクセサリー周辺をきれいに整えすぎない。指の微妙な非対称、関節のしわ、自然な爪、袖口のよれ、縫い目、ゆるい糸、バッグ金具の小傷、ストラップの圧を入れる。
- バッグや靴は無地でシンプルにする。ブランド風の複雑な金具、ロゴ、モノグラム、過剰な装飾、ファンタジー風の形は避ける。
- 4カットすべてを完璧な構図にしない。1枚は少しブレ、逆光、影強め、露出のズレ、端の軽い切れを入れる。別のカットには少し傾き、浅いピント、被写体のオフセンター配置などを入れる。
- 4カットの光と空気感を均一にしない。曇天の柔らかい光、影の多い近接カット、軽い逆光、少し暖かい時間帯などを混ぜる。
- 背景は生活感のある屋外にするが、文字や看板が入り込みすぎる街角には寄せすぎない。コンクリート壁、歩道、手すり、店舗の反射、控えめな街の質感を使い、可読文字や看板は避ける。
- カラーパネルの色名とイメージ語は、画像生成内で入れる。短い英字だけを使い、色名とイメージ語の2段構成にする。
- 初音ミクテーマのAモデル基準では、カラーパネル文言は `AQUA / clear & futuristic`、`BLACK / sharp & cool`、`SILVER / light & clean` を基準にする。

## カラーパネル
- カラーパネルは画像生成内でコーディネート写真の上に重ねる。
- カラーパネルは3つを中央付近に縦に配置し、4つのコーディネート写真の上に重ねる。
- カラーパネル3枚のまとまりは、画像全体の上下左右の中央に見える位置へ配置する。下寄りにしない。
- カラーパネルの色面も縦長3:2で統一する。
- カラーパネルは、余白のある白いカード、大きな色面、下部の色名とイメージ語で構成する。
- 色名は短い英字大文字、イメージ語は短い英字小文字のフレーズにする。
- 色名とイメージ語のフォントは、参考画像のようなエディトリアルなセリフ体・字間広めを基準にする。
- `EC・買い物ガイド` と `トレンド・特集` の記事画像には、中央カラーパネルを重ねない。これらは特定色の提案よりもガイド・特集内容を主役にする。
- `EC・買い物ガイド` と `トレンド・特集` の記事画像では、画像生成プロンプトでも `No color palette cards` を明示し、パネルなしを維持する。

## 確定カラーパネル基準：生成版
- 基準位置は、最新確認済み画像 `/Users/kensuke_kawamura/.codex/generated_images/019f32e4-c89f-7ed3-aab8-56c945181885/ig_0d08a0180a067b65016a4e5d2cc3f881918e292ccce319916.png` の配置とする。
- カラーパネルは中央に3枚縦並びで配置する。
- カラーパネル3枚のグループ全体は、キャンバス中央よりわずかに上へ置く。グループ中心は画像高さの約45%を目安にする。
- 1024x1536の縦画像では、上カードの開始位置を画像上端から約18%、下カードの終了位置を画像上端から約72%に収める。
- 中央カードの中心が、画像全体の中心付近に見えるようにする。
- 見た目が下に寄る場合は、グループ全体を上に移動して中央感を優先する。
- パネルサイズは基準画像から変更しない。1024x1536の縦画像では、各カードはおおよそ幅260px・高さ350pxの見え方を維持し、記事ごとに大きく拡大・縮小しない。
- 3枚のカード間隔も基準画像の見え方を維持する。詰めすぎたり、下カードだけ離したりしない。
- 色面の左右と上部には、細い白余白を残す。余白ゼロにはしない。
- 色面の左右余白と上部余白は同じ太さにする。
- 色面下の白帯には色名とイメージ語を表示する。
- 文字は画像生成で入れる。コード側で後載せする前提にはしない。
- 文字サイズも基準画像から変更しない。色名は大きめの英字大文字、イメージ語はその下の小さめ英字フレーズとし、カード幅に対して余白を残して読みやすく収める。
- 色名・イメージ語は短くする。文字量を増やして、カードサイズや文字サイズが変わる原因を作らない。
